# Solar Teacher Low-Code Hands-on

이 핸즈온에서는 **노트를 넣으면 AI가 피드백과 OX 퀴즈를 만들어 주는 작은 학습 도구**를 직접 실행합니다.

코딩을 많이 하지 않습니다. 대신 n8n이라는 도구에서 “어떤 일이 어떤 순서로 일어나는지”를 눈으로 보면서 AI 에이전트의 기본 구조를 이해합니다.

## 우리가 만들 것

완성된 흐름은 다음과 같습니다.

1. 사용자가 웹 화면에 노트를 입력하거나 이미지/PDF를 업로드합니다.
2. n8n이 요청을 받습니다.
3. 이미지/PDF라면 OCR로 글자를 읽습니다.
4. 참고자료가 있으면 AI에게 함께 전달합니다.
5. AI가 피드백과 OX 퀴즈를 만듭니다.
6. 결과가 다시 웹 화면에 표시됩니다.

[아키텍처 추가 추천] 전체 흐름 그림: `프론트 화면 -> n8n Webhook -> OCR -> RAG-lite -> Upstage Chat -> 결과 화면`

## 이번 실습에서 다루는 개념

| 개념 | 쉬운 설명 |
| --- | --- |
| 프론트엔드 | 사용자가 직접 보는 화면입니다. |
| n8n | 일을 순서대로 연결하는 자동화 도구입니다. |
| Webhook | 외부 요청을 받는 입구입니다. |
| OCR | 이미지나 PDF 안의 글자를 읽어 텍스트로 바꾸는 기술입니다. |
| RAG-lite | AI가 답하기 전에 참고자료를 같이 보게 하는 간단한 방식입니다. |
| LLM | 글을 이해하고 답변을 만드는 AI 모델입니다. |
| Structured output | AI 답변을 XML, JSON처럼 정해진 모양으로 받는 방식입니다. |

## 먼저 알아둘 점

이번 버전은 학습 데이터 저장, 로그인, 회원가입, 피드백 보관함 같은 기능을 제외했습니다. 핵심 목표는 “AI 에이전트가 어떤 순서로 일을 처리하는지”를 이해하는 것입니다.

따라서 결과는 화면에 바로 표시되며, 별도 서비스 DB에 저장하지 않습니다.

# 전체 그림 이해하기

이번 실습은 작은 AI 서비스입니다.

사용자는 웹 화면에 노트를 넣고, n8n은 그 요청을 받아 AI 도구를 호출한 뒤 결과를 다시 화면에 보냅니다.

## 쉬운 비유

식당에 비유하면 다음과 같습니다.

| 실습 요소 | 식당 비유 |
| --- | --- |
| 프론트 화면 | 손님이 주문하는 메뉴판 |
| Webhook | 주문을 받는 카운터 |
| n8n 워크플로우 | 주방의 조리 순서 |
| OCR | 사진 속 글자를 읽는 직원 |
| Upstage Chat API | 요리를 실제로 만드는 요리사 |
| 결과 화면 | 손님에게 나가는 음식 |

## 실제 흐름

1. 프론트 화면에서 사용자가 노트를 입력합니다.
2. 프론트가 n8n Webhook으로 데이터를 보냅니다.
3. n8n이 텍스트인지 파일인지 판단합니다.
4. 파일이면 OCR로 글자를 읽습니다.
5. 참고자료가 있으면 프롬프트에 함께 넣습니다.
6. AI에게 피드백과 퀴즈 생성을 요청합니다.
7. 결과를 프론트가 보기 쉬운 모양으로 정리합니다.

[아키텍처 추가 추천] 위 7단계를 가로 흐름도로 표시하면 좋습니다.

## 왜 n8n을 쓰나요?

n8n은 코드를 많이 쓰지 않고도 여러 도구를 연결할 수 있게 해 줍니다.

이번 실습에서는 n8n을 통해 다음 일을 연결합니다.

- 요청 받기
- 입력 정리하기
- OCR 호출하기
- AI 호출하기
- 결과 정리하기

이 흐름을 눈으로 볼 수 있기 때문에, AI 에이전트가 어떻게 일을 나누어 처리하는지 이해하기 좋습니다.

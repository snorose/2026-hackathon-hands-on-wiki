# 필요한 것 한눈에 보기

아무것도 설치되어 있지 않은 컴퓨터라고 가정하고 시작합니다.

그 전에 먼저, 우리가 왜 n8n을 사용하는지부터 짧게 살펴보겠습니다.

## 업무 자동화와 n8n 소개

미래의 똑똑한 사무실은 어떤 모습일까요?

사람이 파일을 하나씩 열고, 내용을 확인하고, 정리하고, 답장을 쓰는 대신 컴퓨터가 먼저 자료를 읽고 상황을 판단합니다. 필요한 정보만 뽑아내고, 고객이나 동료에게 보낼 답변 초안을 만들고, 다음에 해야 할 일까지 이어서 처리합니다.

이 변화의 핵심은 **자동화가 단순 반복 작업을 넘어서 지능형 의사결정으로 확장되고 있다**는 점입니다.

이번 핸즈온에서 다루는 n8n은 이런 변화를 직접 실습하기 좋은 도구입니다. n8n에서는 업무 흐름을 노드로 연결하고, 그 사이에 OCR, LLM, AI Agent 같은 AI 기능을 붙일 수 있습니다.

## AI 시대 자동화의 패러다임 변화

기존 자동화는 사람이 미리 정해 둔 규칙을 그대로 반복하는 방식에 가까웠습니다. 예를 들어 파일 이름을 바꾸고, 폴더를 옮기고, 정해진 양식의 메일을 보내는 일입니다.

AI 기반 자동화는 한 단계 더 나아갑니다. 문서 내용을 읽고, 의미를 이해하고, 상황에 따라 다른 결정을 내릴 수 있습니다.

| 구분 | 기존 자동화: 전통적 RPA | AI 기반 자동화: AI와 결합된 RPA |
| --- | --- | --- |
| 작업 방식 | 정해진 규칙에 따른 단순 반복 | 상황 판단과 지능형 의사결정 |
| 문서 처리 | 파일 이동, 이름 변경 중심 | 내용을 읽고 이해하여 분류 |
| 고객 응답 | 미리 작성된 템플릿 전송 | 맞춤형 답변을 실시간 생성 |
| 데이터 분석 | 단순 집계와 차트 생성 | 패턴 분석과 인사이트 도출 |

전통적 RPA 워크플로우는 보통 다음처럼 움직입니다.

1. 입력 파일을 받습니다.
2. 파일 이름을 일괄 수정합니다.
3. 지정된 폴더로 이동합니다.
4. 파일 이름을 시트에 기록합니다.

AI 기반 자동화 워크플로우는 다음처럼 확장됩니다.

1. 입력 문서나 이미지를 받습니다.
2. AI가 어떤 처리가 필요한지 판단합니다.
3. 이미지를 디지털 문서로 변환합니다.
4. 긴 내용을 요약하거나 필요한 정보를 추출합니다.
5. 결과를 다음 업무 흐름으로 넘깁니다.

:::info RPA란?
RPA(Robotic Process Automation)는 "로봇 프로세스 자동화"라는 뜻입니다. 컴퓨터가 사람이 하던 반복 업무를 대신 수행하는 기술입니다. 마치 24시간 일하는 디지털 직원처럼 클릭, 타이핑, 복사, 파일 이동 같은 작업을 자동으로 처리합니다.
:::

## 3가지 핵심 AI 자동화 기술

이번 핸즈온은 단순히 n8n 버튼을 눌러 보는 수업이 아닙니다. AI 시대 자동화에서 자주 쓰이는 3가지 기술을 작은 프로젝트 안에서 함께 경험합니다.

| 기술 | 무엇을 하나요? | 이번 실습에서의 예 |
| --- | --- | --- |
| Document AI | 이미지나 PDF 안의 글자를 읽고 구조화합니다. | 업로드한 노트 이미지에서 텍스트를 추출합니다. |
| LLM 챗봇 | 글을 이해하고 사람에게 필요한 답변을 만듭니다. | 노트 내용을 바탕으로 피드백과 OX 퀴즈를 생성합니다. |
| Agent AI | 상황을 보고 필요한 도구와 흐름을 선택합니다. | 입력 종류에 따라 OCR, 참고자료, 답변 생성 흐름을 연결합니다. |

여기서 중요한 점은 세 기술이 따로 떨어져 있지 않다는 것입니다. 실제 자동화에서는 문서를 읽고, 내용을 이해하고, 다음 행동을 선택하는 과정이 하나의 워크플로우로 이어집니다.

## n8n의 특별함

n8n은 워크플로우 자동화 도구입니다. 어떤 일이 발생했을 때, 어떤 순서로 어떤 도구를 실행할지 시각적으로 연결할 수 있습니다.

Zapier나 Make 같은 도구도 비슷한 자동화를 만들 수 있습니다. 하지만 n8n은 AI와 자동화를 함께 다루는 실습에서 특히 장점이 큽니다.

| 구분 | 세부 항목 | 전통적 자동화 도구(Zapier/Make) | n8n |
| --- | --- | --- | --- |
| 공통점 | 핵심 기능 | 노코드 워크플로우 구축, 다양한 앱/서비스 연동, 트리거 기반 자동화, API 통합 지원 | 노코드 워크플로우 구축, 다양한 앱/서비스 연동, 트리거 기반 자동화, API 통합 지원 |
| 차이점 | 비용 모델 | 사용량이 늘수록 비용이 커질 수 있습니다. | 오픈소스로 직접 실행할 수 있습니다. |
| 차이점 | 사용량 확장성 | 월 사용량 제한이나 추가 요금이 있을 수 있습니다. | 직접 실행하면 실습 범위 안에서 자유롭게 사용할 수 있습니다. |
| 차이점 | 데이터 소유권 | SaaS 업체 환경에 의존합니다. | 내 컴퓨터나 내가 관리하는 서버에서 실행할 수 있습니다. |
| 차이점 | 보안 통제 | 외부 서비스 정책에 의존합니다. | 데이터를 어디에서 처리할지 직접 통제할 수 있습니다. |
| 차이점 | AI 기능 | 기본 자동화 중심입니다. | AI Agent 워크플로우를 만들기 좋습니다. |

n8n의 핵심 장점은 다음과 같습니다.

- LangChain 기반 AI 기능을 활용해 Agent 형태의 흐름을 설계하기 좋습니다.
- 사용량이 늘어도 실습 환경에서는 추가 과금 걱정 없이 반복해서 실험할 수 있습니다.
- 워크플로우와 데이터를 직접 관리할 수 있어 구조를 이해하기 쉽습니다.

## 왜 지금 n8n을 배워야 할까요?

AI 도구를 잘 쓰는 것과 AI가 들어간 업무 흐름을 설계하는 것은 다릅니다.

앞으로는 "AI에게 질문을 잘하는 사람"을 넘어, **AI가 언제 어떤 도구를 사용해야 하는지 설계할 수 있는 사람**이 더 큰 생산성을 만들 수 있습니다. n8n은 이 감각을 배우기에 좋습니다. 코드를 많이 쓰지 않아도 트리거, 조건 분기, API 호출, 문서 처리, LLM 호출, 응답 정리까지 한눈에 볼 수 있기 때문입니다.

이 핸즈온을 마치면 다음 역량을 얻게 됩니다.

| 영역 | 기대 성과 |
| --- | --- |
| 기술적 역량 | n8n 노코드 도구 이해, AI Agent 흐름 설계, RAG-lite 구성, 자동화 시스템 구조 파악 |
| 비즈니스 가치 | 반복 업무 감소, 24시간 자동 처리 가능성 이해, 데이터 기반 의사결정, 개발 시간과 비용 절약 |

## 점진적 개선으로 접근하기

처음부터 완벽한 자동화 시스템을 만들 필요는 없습니다. 실무 자동화는 보통 작게 시작해서 점점 고도화합니다.

### 1단계: 간단한 시작

첫 워크플로우에서는 작동 확인에 집중합니다.

- 단순한 문서 분류를 테스트합니다.
- 필요한 정보를 한두 개만 추출합니다.
- 노드 연결과 데이터 흐름을 이해합니다.
- 결과가 제대로 나오는지 확인합니다.

### 2단계: 실무 고도화

기본 흐름이 작동하면 조금씩 기능을 더합니다.

- 폴더 변화를 감지해 자동 실행합니다.
- 여러 PDF를 한 번에 처리합니다.
- 추출 스키마를 저장하고 재사용합니다.
- 완료된 파일을 자동으로 아카이브합니다.

:::tip 핵심 전략
처음에는 간단히 시작해서 작동만 확인해도 충분합니다. 그다음 차근차근 기능을 더하며 고도화된 자동화 시스템으로 발전시키면 됩니다.
:::

## 요리를 배우듯 단계별로 익히기

자동화를 배우는 과정은 요리를 배우는 과정과 비슷합니다. 먼저 재료와 도구를 알고, 기본 조리법을 익힌 뒤, 실무에서 바로 쓸 수 있는 대표 요리를 하나씩 만들어 봅니다.

| 단계 | 내용 | 이 문서에서의 위치 |
| --- | --- | --- |
| 기본 개념 | 자동화 도구와 실행 환경을 준비합니다. | 처음 환경 준비하기, 실습 폴더 이해하기 |
| 문서 처리 자동화 | PDF와 이미지에서 텍스트를 추출합니다. | OCR 워크플로우 |
| 언어모델 활용과 의미 검색 | 참고자료를 함께 넣어 더 정확한 답변을 만듭니다. | RAG-lite |
| 에이전트 활용 | 입력 상태에 따라 여러 도구를 연결합니다. | n8n 워크플로우 뜯어보기 |
| 유지보수와 고도화 | 에러를 해결하고 실제 서비스로 확장합니다. | 에러 해결하기, 마무리와 확장 |

이제 큰 그림을 봤으니, 실제 환경 준비를 시작하겠습니다.

## 필요한 도구

| 필요한 것 | 왜 필요한가요? |
| --- | --- |
| 웹 브라우저 | n8n과 실습 화면을 열기 위해 필요합니다. |
| 터미널 | 명령어를 실행하는 창입니다. |
| Docker Desktop | n8n을 쉽게 실행하기 위해 필요합니다. |
| 텍스트 편집기 | `.env` 파일을 수정할 때 사용합니다. |
| Upstage API Key | OCR과 AI 답변 생성을 사용하기 위한 열쇠입니다. |
| 실습 폴더 | 프론트 화면과 n8n 워크플로우 파일이 들어 있습니다. |

## n8n을 따로 설치하지 않습니다

n8n은 직접 설치하지 않습니다. Docker Desktop을 설치한 뒤, 명령어 하나로 n8n을 실행합니다.

```bash
docker compose up -d
```

이 명령어는 “n8n을 담은 실행 상자를 내려받고, 백그라운드에서 켜 주세요”라는 뜻입니다.

## 전체 준비 순서

1. 터미널을 엽니다.
2. Docker Desktop을 설치하고 실행합니다.
3. Upstage API Key를 준비합니다.
4. 실습 폴더를 준비합니다.
5. `.env` 파일에 API Key를 넣습니다.
6. n8n과 프론트 화면을 실행합니다.

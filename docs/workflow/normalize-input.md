# 입력 정리하기

`Normalize Input` 노드는 프론트에서 받은 데이터를 n8n이 쓰기 좋은 형태로 정리합니다.

## 하는 일

1. 노트 텍스트가 있는지 확인합니다.
2. 참고자료 텍스트를 꺼냅니다.
3. 파일이 있으면 base64를 n8n 내부 binary로 바꿉니다.
4. 다음 노드가 판단하기 쉬운 값을 만듭니다.

## 왜 정리 단계가 필요할까요?

사용자는 텍스트로 입력할 수도 있고, 이미지/PDF 파일을 업로드할 수도 있습니다.

두 입력 방식은 모양이 다릅니다.

그래서 먼저 하나의 기준으로 정리해야 뒤쪽 노드들이 단순해집니다.

## 만들어지는 값

| 값 | 의미 |
| --- | --- |
| `raw_text` | 사용자가 직접 입력한 노트 텍스트입니다. |
| `reference_text` | 참고자료 텍스트입니다. |
| `hasText` | 텍스트 노트가 있는지 표시합니다. |
| `hasDocument` | 업로드 파일이 있는지 표시합니다. |
